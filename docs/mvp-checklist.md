# MVP 체크리스트

## 완료

- 백엔드 패키지 루트가 `com.aimentor`로 정렬되었고, 기능 패키지가 `common`, `domain.*`, `external.*`로 분리되었습니다.
- 프론트엔드 CRUD API 클라이언트가 `/api/v1/profiles/*` 하위의 백엔드 버전 경로를 호출하도록 정리되었습니다.
- 프론트엔드 인증 흐름이 백엔드 토큰 응답을 정규화하고, 인증 세션을 저장하며, 보호된 API 요청에 `Authorization` 헤더를 전송합니다.
- 프론트엔드 이력서, 자기소개서, 채용공고 폼이 백엔드 요청/응답 계약과 맞는 DTO 필드명을 사용합니다.
- 백엔드 헬스 체크 엔드포인트가 `/api/v1/health`로 정렬되었고 공개 접근이 가능합니다.
- Vite 개발 서버가 로컬 연동을 위해 `/api` 요청을 `http://localhost:8080`으로 프록시합니다.
- 활성 세션이 없을 때 면접 진행 페이지에 직접 진입해도 더 이상 빈 화면이 표시되지 않습니다.

## 미완료

- 프론트엔드 면접 흐름은 아직 스토어 중심이며 `/api/v1/interviews/sessions`, `/answers`, `/end`, `/report`를 호출하지 않습니다.
- 대시보드와 면접 결과 페이지는 아직 백엔드 세션/리포트 API 대신 로컬 플레이스홀더 데이터를 사용합니다.
- 프론트엔드 인증 생명주기가 완성되지 않았습니다. 리프레시 토큰 갱신, 로그아웃 API 호출, 새로고침 시 세션 복원, 라우트 가드가 아직 구현되지 않았습니다.
- 프론트엔드 라우트 모델은 아직 세션 ID 중심 구조가 아닙니다. 면접 진행/결과 라우트에 ID가 없어 딥링크와 새로고침 복구가 약합니다.
- 백엔드 DTO 네이밍이 완전히 통일되지는 않았습니다. 프로필 요청은 `*UpsertRequest`, 면접 요청은 액션 중심 이름을 사용하고 있어 동작은 가능하지만 일관성은 부족합니다.
- 문서가 아직 부분적입니다. `frontend/README.md`는 기본 Vite 템플릿 상태이고, `VITE_USE_API_STUB`와 API 기대값을 설명하는 프론트엔드 `.env.example`도 없습니다.

## 다음 작업

1. 면접 Zustand 흐름을 실제 면접 세션 API로 교체하고 `sessionId` 기반 라우팅으로 전환합니다.
2. 프론트엔드 인증 초기화, 리프레시/로그아웃 처리, 보호된 라우트 가드를 추가합니다.
3. 대시보드와 결과 플레이스홀더 데이터를 백엔드 세션 이력 및 리포트 엔드포인트로 교체합니다.
4. 프론트엔드 실행 문서와 스텁 모드/백엔드 API 모드를 위한 `.env.example`을 추가합니다.
